# Симуляция бюджета фрилансера: Параметризованное тестирование в Maven (JUnit 5)

Учебный проект, посвященный моделированию бизнес-логики на Java (циклы и условия) и углубленной автоматизации тестирования. Ключевые особенности проекта — организация кода по пакетам, управление зависимостями через Maven и использование **параметризованных автотестов (Data-Driven Testing)** с помощью CSV-файлов.

## 🎯 Цели и задачи проекта
* **Пакетная структура**: организация классов в соответствии со стандартами Java-разработки (размещение кода и тестов в пакете `ru.netology.services`).
* **Параметризованное тестирование**: уход от дублирования кода в тестах (типа *один тест — один метод*) за счет использования аннотации `@ParameterizedTest` и внешних CSV-файлов с тестовыми данными.
* **Моделирование логики**: написание алгоритма, симулирующего помесячный расчет финансового состояния на протяжении года с использованием циклов.
* **Инфраструктурная чистота**: работа в профессиональной среде IntelliJ IDEA, жесткая изоляция репозитория от мусорных файлов (`.idea`, `target`, `*.iml`) через `.gitignore`.

## 💻 Суть задачи (Бизнес-логика симулятора)
Программа моделирует финансовый год фрилансера на основе трех параметров: **income** (доход за месяц работы), **expense** (обязательные траты) и **threshold** (порог накоплений для принятия решения об отдыхе).

**Логика выбора на следующий месяц:**
1. Если на счету **есть хотя бы `threshold` денег**, фрилансер выбирает **отдых**:
   * Баланс уменьшается на обязательные траты `expense`.
   * Остаток накоплений дополнительно уменьшается в три раза (траты на отдых).
2. Если денег меньше `threshold`, фрилансер выбирает **работу**:
   * Баланс увеличивается на `income` и уменьшается на `expense`.

### 📋 Пример для самопроверки:
* `income` = 10 000 руб., `expense` = 3 000 руб., `threshold` = 20 000 руб.
* *Ожидаемый результат*: **3 месяца отдыха в году**.

## 🧪 Реализация автотестов и CSV-параметризация
Вместо написания отдельных методов под каждый тестовый сценарий, в проекте реализован один гибкий параметризованный тест в классе `VacationServiceTest`.

* Данные для тестов (входные параметры и ожидаемый результат) вынесены в отдельный файл формата **CSV**.
* Тестовый движок JUnit 5 автоматически считывает строки из файла, подставляет их в аргументы метода и запускает тест для каждого набора данных.

Полный цикл сборки, статического анализа и выполнения параметризованных тестов запускается через терминал:
```bash
mvn clean test
```
*Сборка гарантированно завершается успешно (`BUILD SUCCESS`), а все тесты проходят.*
